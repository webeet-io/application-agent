# Task Briefs F1-F4

See also: [overview.md](../overview.md)

## F1-Backend-1

Feature:
`F1 Foundation Skeleton`

Role:
`Backend`

What this task is:
Create the backend application skeleton in `apps/api` with clear transport, application, and adapter entry structure.

Why it exists:
Later backend, AI, and MCP tasks need a stable code home before feature logic begins.

Likely area:
- `apps/api/`
- `packages/domain/`
- `packages/shared/`

Depends on:
- none

Can start in parallel with:
- `F1-Frontend-1`
- `F1-Cloud-1`
- `F1-Test-1`

Done when:
- backend app entry exists
- package wiring is valid
- clear placeholder structure exists for transport, application services, and adapters

Out of scope:
- real business feature logic
